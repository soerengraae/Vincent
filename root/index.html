How it works

From barcode to beautiful

The whole pipeline runs on the device — no companion app, no Bluetooth handshake, nothing on your phone. Just a microcontroller doing its job.

Scan
UART barcode scanner reads the UPC off the sleeve
Query
HTTP GET to Discogs REST API with OAuth token
Parse
cJSON extracts title, artist, year, label from response
Display
4-color e-ink renders the record with custom bitmap fonts
Auto-confirm

Confirmed after 20 seconds — the record is added to your Discogs collection via HTTP POST (201 on success). Not right? Tap the shelf: the ADXL345 accelerometer catches the vibration and cancels before anything is saved.

Software

Written in C, by choice

No Arduino. No Node. The whole stack lives on the chip — written in C against ESP-IDF, with FreeRTOS coordinating tasks.

ESP-IDF Espressif's official IoT Development Framework. Ported from an earlier Arduino prototype to gain full control over SPI, memory, and task scheduling. Core
FreeRTOS Real-time OS underneath ESP-IDF. Concurrent tasks for scanner polling, API calls, display rendering, and accelerometer monitoring run side-by-side. Core
esp_http_client HTTP/S client for communicating with the Discogs REST API. Handles OAuth 1.0a personal token auth for both GET (search) and POST (collection add). Networking
cJSON Lightweight JSON parser. Extracts title, artist, year, label, and format from Discogs API responses with minimal heap overhead. Parsing
Discogs API REST API queried by UPC barcode string. Returns release metadata, matched against the user's collection. POST /users/{username}/collection adds confirmed records. External
Custom Bitmap Fonts Hand-converted Roboto Bold at 4 point sizes, stored as C header arrays. Rendered directly to the SPI frame buffer. Includes vinyl disc count glyphs. Display
